# Class Design Rules v1

## Core Philosophy

Classes should evolve progressively.

Advancement should improve:

- Battlefield scale.
- Specialization.
- Tactical identity.

Advancement should not completely replace gameplay identity.

---

## Core Rules

- Each branch should specialize in 1 to 2 core mechanics.
- Avoid overloading a single skill with too many effects.
- Higher-tier jobs should improve battlefield influence.
- Skill evolution should feel natural and readable.
- Classes should remain identifiable during progression.

---

## Skill Evolution Rules

Lower-tier skills:

- Simple.
- Focused.
- Single-purpose.

Mid-tier skills:

- Improved scale.
- Stronger interaction.
- Secondary effects.

High-tier skills:

- Battlefield influence.
- Stronger synergy.
- Wider impact.

Legendary-tier skills:

- Define encounter presence.
- Visually recognizable.
- Strategically important.

## Buff / Debuff Rules

- Avoid excessive buff stacking.
- Avoid giving every class every status type.
- Buffs and debuffs should reinforce class identity.
- Important statuses should remain readable on mobile.

---

## Combat Readability Rules

- Avoid excessive visual spam.
- Avoid constant full-screen effects.
- Preserve portrait mobile readability.
- Battlefield visibility has highest priority.

---

## Hidden Class Rules

- 重劍者

- 見習女巫

---

## Development Guardrails

Codex must not:

- Create new class branches automatically.
- Change advancement structures.
- Add excessive mechanics without approval.
- Ignore existing class identities.
- Convert the game into high-number combat design.
