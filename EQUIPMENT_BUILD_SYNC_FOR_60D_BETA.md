## 2. 裝備取得核心循環

核心循環：

```text
遠征
→ 怪物素材／場景素材
→ 工坊加工一次
→ 基礎加工品
→ 鐵匠
→ 裝備
```

正式規則：

- 怪物不直接掉可穿戴成品裝。
- 普通怪：掉落一般素材與金幣。
- 精英怪：掉落高級素材、基礎加工品來源、殘缺裝素材；殘缺裝來源以精英怪為主。
- Boss：掉落 Boss 素材與 Boss 核心素材。
- `MONSTER_DROP_LIST_CH01_05.md` 只整理 Chapter 01～05 怪物素材掉落，不包含殘缺裝或可穿戴裝備。
- `material_usage_master_database.md` 只做素材用途分類，不建立數值、掉率或配方；素材名稱以 `monster_material_master_database.md` 為準。

## 3. 裝備品質定位

裝備品質：

- 白裝：基礎過渡裝。
- 綠裝：小型 Build 過渡裝。
- 藍裝：正式 Build 起點。
- 紫裝：核心 Build 裝。
- 暗金：Boss 榮耀裝。

節奏理解：

- 白裝、綠裝不應成為長期追求核心。
- 藍裝承接成熟角色成長與正式 Build 起步。
- 紫裝提供核心 Build 方向。
- 暗金提供 Boss 記憶、章節榮耀與固定特殊效果。

## 4. 裝備製作流程

### 白裝／綠裝／藍裝

正式流程：

```text
怪物素材／場景素材
→ 工坊加工一次
→ 基礎加工品
→ 鐵匠製作裝備
```

規則：

- 白裝、綠裝、藍裝可由 Codex 依規則與素材資料庫批次生成。
- 白裝、綠裝、藍裝不需要第二層加工。
- 白裝、綠裝、藍裝不使用文明加工品。

### 紫裝

正式流程：

```text
基礎加工品
+
殘缺裝素材
→ 鐵匠製作紫裝
```

規則：

- 紫裝不可自動大量生成，需先依設計討論定案。
- 紫裝需引用 `BROKEN_EQUIPMENT_DATABASE.md`。
- 紫裝不需要文明加工品。
- 紫裝不使用文明素材作為封測正式需求。

### 暗金裝

正式流程：

```text
基礎加工品
+
Boss素材
+
Boss核心素材
→ 鐵匠製作暗金裝
```

規則：

- 暗金裝不可自動大量生成，需先依設計討論定案。
- 暗金裝固定 Boss 效果。
- 暗金裝不採隨機詞條。
- 暗金裝不採選擇詞綴。
- 暗金裝是章節 Boss 榮耀追求，不是數值膨脹裝。
- 暗金裝不需要文明加工品。
- 暗金裝不使用文明素材作為封測正式需求。

## 5. 紫裝與殘缺裝規則摘要

殘缺裝規則摘要：

- 殘缺裝由 `BROKEN_EQUIPMENT_DATABASE.md` 管理。
- 殘缺裝不是 `monster_material_master_database.md` 的一般素材。
- 殘缺裝不是成品裝備，不可直接裝備。
- 殘缺裝定位為紫裝核心素材。
- 每章殘缺裝 2～4 件。
- 每章至少 1 件可製作全職業通用飾品。
- 未取得殘缺裝支援的職業，仍應可透過白裝、綠裝、藍裝持續成長。

## 6. 封測暫不採用

以下內容不可作為封測 Current Rule：

- 文明加工品
- 高階文明素材
- 王國精鋼
- 皇家鍛鐵
- 王國紋章鋼片
- 第二層加工鏈
- 第三層加工鏈
- 白裝、綠裝、藍裝需要兩次以上加工
- 紫裝需要文明素材或文明加工品
- 暗金需要文明素材或文明加工品
- Boss 裝由高階基底 + Boss素材 + Boss核心素材 + 文明素材鍛造

---

## 7. 60 天封測節奏定位

節奏方向：

- 紫裝目標取得：1～2 天。
- Boss／暗金裝目標取得：2～5 天，平均 3～4 天。

封測成長理解：

- 封測到第 5 章／6 轉時，藍裝為主要成熟裝。
- 紫裝為核心 Build 追求。
- 暗金為章節 Boss 榮耀追求。
- 不要求玩家每章每職業都做完紫裝。
- 主線升級不應要求玩家先完成所有紫裝或暗金裝。
- 裝備追求要服務 60 天封測節奏，而不是無限刷寶。

## 8. 禁止事項

不要：

- 不要把殘缺裝加入 `monster_material_master_database.md`。
- 不要讓怪物掉白裝、綠裝、藍裝、紫裝、暗金裝。
- 不要生成完整裝備表。
- 不要新增怪物素材名稱。
- 不要自行改名素材、殘缺裝或裝備品質。
- 不要讓裝備追求變成大量同職業同品質重複裝備。

## 9. 一句話結論

```text
怪物與場景提供素材
→ 工坊加工一次成基礎加工品
→ 鐵匠製作裝備
→ 藍裝成熟
→ 紫裝建立核心 Build
→ 暗金形成 Boss 榮耀追求
```

裝備系統不走大量掉裝、不走多層加工膨脹、不走隨機詞條膨脹。封測重點是讓玩家在 Chapter 01～05 期間持續有清楚、可達成、可辨識的 Build 目標。
